; ============================================================================
; KNOWN LIMITATION: Defective matrices with non-zero eigenvalues are not detected.
;
; The wrapper-level zero-column scan in realEigenvectors / complexEigenvectors
; catches defective inputs only when calculateEigenvectors leaves a column all
; zero (e.g. the augmented system fails fully singular). This works for nilpotent
; cases like [[0,0],[1,0]] where the eigenvalue is 0 and (A - 0*I) is fully
; singular.
;
; For Jordan blocks with non-zero eigenvalue (e.g. [[2,1,0],[0,2,1],[0,0,2]]),
; the augmented system finds non-zero pseudo-eigenvectors that do not satisfy
; A*v = lambda*v. The columns are non-zero so the scan does not trigger, and
; EC=0 is returned with mathematically wrong vectors.
;
; Detecting this would require a residual check (verify ||A*v_k - lambda_k*v_k||
; against a scaled tolerance) added to the wrappers, similar to verifySqrtMatrix.
; That is a viable future improvement but adds ~60 lines per wrapper for two
; edge cases. Re-enable these tests if/when that check is implemented.
